# Superseded Documentation Files

This folder contains original documentation files that have been reorganized into the new modular documentation structure.

## Archived Files

| File | Original Purpose | Replaced By |
|---|---|---|
| `architecture.original.md` | Combined architecture and protocol specification | `OVERVIEW.md`, `PROTOCOL.md`, `SEMANTIC.md`, `DELEGATION.md`, `SECURITY.md` |
| `monorepo.original.md` | Monorepo migration guide and package structure | `MONOREPO.md` |

## New Documentation Structure

The documentation has been reorganized into focused, purpose-driven files:

| File | Purpose |
|---|---|
| `README.md` | Clear, concise overview and entry point |
| `OVERVIEW.md` | Definitive architectural and protocol specification |
| `PROTOCOL.md` | P2P networking, DHT, and communication protocols |
| `SEMANTIC.md` | Embedding models, relational matching, semantic geometry |
| `DELEGATION.md` | Supernode architecture and delegation protocol |
| `SECURITY.md` | Threat model, safety, privacy, authenticity |
| `SOCIAL.md` | Social network layer (posts, feeds, communities) |
| `MONOREPO.md` | Modular architecture and package structure |
| `GETTING_STARTED.md` | Setup, usage, and local testing |
| `ROADMAP.md` | Development phases and timeline |

## Why Reorganized?

The original documentation files were:
- **Too large**: README.md was 1000+ lines, difficult to navigate
- **Mixed concerns**: Architecture, protocol, security, and social features combined
- **Hard to reference**: No clear separation for linking to specific topics

The new structure:
- **Focused files**: Each document has a clear, single purpose
- **Easier navigation**: Users can find relevant information quickly
- **Better maintainability**: Changes to one area don't affect unrelated docs
- **No cross-links**: Each file stands alone as a complete specification
